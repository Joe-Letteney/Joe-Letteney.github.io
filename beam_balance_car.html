Beam Balance Car Project

Goal

The goal of this project was to create a small car that could travel a set distance as fast as possible while balancing a one-foot beam on top. This involved several design challenges, including determining the maximum possible acceleration, writing a PID controller that could smoothly translate the desired velocity profile into motor speeds, and designing a car capable of accomplishing all of this.

Calculation

The first step in this project was to determine the maximum theoretical acceleration that the car could achieve before the bar tips over. Using moment balance about the edge of the bar, we determined it to be 1/12 of the acceleration due to gravity, or 32 in/s². Using this, we calculated our velocity given the maximum possible distance of ten feet. We then tested the motor we were given for the project and determined that its max speed was 235 rev/min. Knowing this, we determined that to achieve our max acceleration, we would need wheels approximately 5 inches in diameter.

Simulation

The next step was to test our calculated values using SolidWorks’ Motion Analysis tool. To start, I set up a basic simulation with the bar on top of a plate, applied an increasing acceleration profile, and then plotted the reaction forces holding the bar on top. I found the point where one of them went to zero, indicating that the bar would become unstable at that point. As you can see in the figure below, this agrees with our theoretical value.

Next, I set up a more detailed simulation to test our theoretical velocity profile. I modeled a simple cart in SolidWorks and then applied the velocity profile, confirming that it did indeed go the intended distance and that the bar remained upright throughout.

CAD Design

To achieve the features described above, I modeled a small car in OnShape. One of the major difficulties in this project was accounting for any unevenness in the floor, so I opted for the maximum allowed length, which was one foot.

Controller Code

To achieve the velocity profile that we determined above, our team wrote a PID controller. This allowed us to achieve a smooth velocity profile, which is essential for this problem because dramatic shifts in the elevation profile translate to large accelerations, which would cause the bar to fall over.
